04 — Project Gist & Summary

What is Traveloop?

Traveloop is a travel planning SPA (Single Page Application) built entirely in one React JSX file (~2,170 lines). It simulates a full-featured trip management platform — complete with auth, dashboard, trip CRUD, itinerary builder, budget tracking, community feed, invoicing, notes, packing checklist, and an admin panel — all without any real backend or database.


It is a prototype / demo: all data is hardcoded in a MOCK object, auth is simulated, and "saving" does nothing persistent. Ideal as a UI showcase, interview demo, or starting scaffold.

Architecture Choices

Manual router: No React Router. Screen switching is a switch(screen) statement driven by useState.

CSS-in-JS (string): All styles live in a template literal injected into a <style> tag — no Tailwind, no CSS modules, no styled-components.

No global state: Props are drilled manually. setScreen / toast / selectedTrip are passed as props from root to children.

Role-based access: Admin screens only visible if user.role === "admin".
